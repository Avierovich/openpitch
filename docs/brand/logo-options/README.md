# OpenPitch Logo Options

These five concepts are vector-first logo directions for an open-source, MCP-native developer tool. They are designed to work in GitHub READMEs, MCP directories, dark/light UI surfaces, and small app icons.

## Options

1. **Signal Pitch** (`01-signal-pitch.svg`) - pitch-card + freshness signal. Best for the "fresh startup intelligence" story.
2. **Receipt Node** (`02-receipt-node.svg`) - receipt + graph nodes. Best for provenance, citations, and "show the receipts."
3. **Agent Bridge** (`03-agent-bridge.svg`) - OP monogram + connection frame. Best for MCP/A2A positioning.
4. **Git Radar** (`04-git-radar.svg`) - radar + git-history lines. Best for freshness and public audit trail.
5. **Terminal Proof** (`05-terminal-proof.svg`) - terminal + verified trend. Best for developer-first GitHub launch assets.

## Selected Direction

The selected direction is **Terminal Proof** (`05-terminal-proof.svg`).

Why: it best matches the current launch surface: developer-first, CLI/MCP oriented, and immediately legible in a GitHub README. It says "ask your agent, get sourced numbers" without over-indexing on generic finance imagery.

Keep **Agent Bridge** as a future icon-system or MCP-directory variant if the brand later leans harder into A2A infrastructure.
